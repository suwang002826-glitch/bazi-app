# 紫金山天文台历法数据合作咨询邮件草稿

收件人建议：lisuan@pmo.ac.cn

主题：关于历法、二十四节气及农历数据授权合作的咨询

中国科学院紫金山天文台历算团队老师您好：

您好！我们正在开发一款面向传统文化学习与命理排盘场景的八字排盘小程序，并计划在后续升级为 App 产品。产品核心需要严谨、可追溯的历法底座，因此希望咨询贵台历法数据的合作与授权方式。

我们了解到紫金山天文台长期承担历书天文、年历编算及相关历法研究工作，也是《农历的编算和颁行》等国家标准的重要起草单位之一。为了避免使用非权威数据导致农历、节气、朔望及四柱排盘出现偏差，我们希望在产品测试和后续商业化阶段，尽可能采用贵台权威数据或经贵台认可的数据口径。

我们希望咨询的数据范围包括：

1. 公历与农历转换数据，含闰月、大小月、朔望月信息；
2. 二十四节气精确交节时刻，最好精确到分钟或秒；
3. 每日干支及历法基础字段；
4. 可用于校验排盘的大范围年份数据，例如 1900-2100 年；
5. 数据版本、更新周期、引用规范和授权边界。

也想请教贵台目前是否提供以下合作方式：

1. 结构化数据授权，例如 JSON、CSV、Excel 或数据库文件；
2. API 或服务端查询接口；
3. 电子版《中国天文年历》或历法数据的授权使用；
4. 商业 App / 小程序使用贵台历法数据的授权流程、费用标准及署名要求；
5. 如暂无直接授权 API，是否可以基于公开出版物或贵台公开资料建立内部校验库。

我们当前产品仍处于测试版阶段，短期只会用于小范围内测。为了保证专业性，我们希望先建立可追溯的数据来源和校验流程；如后续产品正式上线并商业化，也愿意按照贵台要求办理正式数据授权或合作手续。

如方便，烦请告知我们应联系的具体部门或老师，以及需要提交的材料。我们也可以提供产品说明、技术方案和数据使用场景供贵台评估。

感谢您在百忙之中阅读此邮件，期待您的指导与回复。
