# Codex 项目规则

本仓库是“玄问 AI / Bazi-app”微信小程序项目。Codex 在本仓库工作时必须遵守以下规则。

## 协作规则

- 主要开发目录是 `code/`。
- `main` 分支只放稳定版本，不直接做实验性修改。
- 每个新功能或修复都先建 Issue，再开独立分支。
- 修改完成后通过 Pull Request 合并，至少由另一位协作者检查。
- 不覆盖同伴已有代码，不删除未确认用途的文件。
- 每次只处理当前任务，不顺手重构无关模块。

## 产品边界

- 当前第一版继续基于微信小程序开发。
- 当前产品包含八字、六爻、奇门、命例档案等模块。
- 任何断语、解释、建议都必须保持传统文化学习和参考口径，不做医疗、法律、投资等重大决策承诺。
- 用户界面必须保持简体中文。

## 八字规则

- 排盘规则变更必须先更新 `docs/排盘规则.md`。
- 准确性问题必须记录到 `docs/验收样例.md` 或对应 Issue。
- 遇到流派差异，先记录为“待确认”，不要直接写死为唯一标准。
- 正式上线前，需要用真实样例复核节气换月、子时换日、真太阳时、闰月、大运起运等关键边界。

## 验证要求

- 微信小程序代码改动后，优先使用微信开发者工具导入 `code/` 目录预览。
- 改动八字、六爻、奇门核心逻辑时，必须说明验证样例。
- 文档改动至少确认 Markdown 能正常阅读。
