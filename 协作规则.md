# 协作规则

## 固定流程

每个任务都按这个顺序推进：

```text
创建 Issue
指定负责人
创建任务分支
让 Codex 开发
本地运行检查
创建 Pull Request
另一人验收
合并到 main
```

## 分支命名

- 输入页面：`feature/input-form`
- 排盘核心：`feature/bazi-core`
- 保存记录：`feature/save-records`
- 分享图片：`feature/share-image`
- 修复问题：`fix/问题简述`

## Pull Request 规则

- 每个 Pull Request 只做一件事。
- 标题写清楚完成了什么。
- 描述里写验收方式。
- 另一个人检查后再合并。
- 合并后删除任务分支。

## 每日同步

每天只需要同步三件事：

1. 今天做哪个 Issue？
2. 谁负责验收？
3. 哪些内容已经合并到 `main`？

## 两人分工

产品负责人负责：

- 页面结构
- 中文文案
- 操作流程
- 手机端体验
- 用户反馈记录

规则负责人负责：

- 排盘规则
- 验收样例
- 样例对照
- 流派差异记录
- 排盘准确性验收
