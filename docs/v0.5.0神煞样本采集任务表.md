# v0.5.0 神煞样本采集任务表
## 任务基本信息
- 目标版本：v0.5.0 神煞功能版
- 任务目标：收集10个核心常用神煞的问真八字标准样本，为神煞计算引擎开发提供唯一验收依据
- 核心原则：**所有神煞结果必须100%来自问真八字实际排盘显示，禁止使用传统命理资料自行推断规则、补全结果**
- 预计工作量：单人约1-2小时可完成

---

## 采集流程
1. 打开问真八字（网页版/APP均可），输入出生信息完成排盘
2. 找到排盘结果中的神煞显示区域，完整截图保存（截图需清晰看到四柱和神煞命中位置）
3. 记录输入的出生信息、真太阳时/夏令时开关状态
4. 记录问真显示的完整四柱、神煞命中的柱位和对应地支
5. 将信息填入样本模板，截图保存到`docs/shensha-samples/`目录下

---

## 采集顺序
按优先级从高到低采集，单个神煞样本校验通过后再采集下一个：
**天乙贵人 → 文昌 → 驿马 → 桃花 → 华盖 → 羊刃 → 太极贵人 → 将星 → 天德/月德**

## 样本字段说明
所有样本需严格按照以下格式填写，字段不可缺省：
```js
{
  spiritName: "神煞名称，必须和上表完全一致",
  rule: {
    basis: "问真显示该神煞的查法依据，例如「以日干查，寅午戌见午为桃花」，必须和问真规则说明一致",
    calculation: "该神煞在问真中的命中位置规则，例如「日支为午，命中年支戌，为桃花在年柱」，只记录实际命中的情况，不自行外推",
    confirmedBy: "问真八字" // 固定值
  },
  cases: [
    {
      caseId: "wz-神煞拼音缩写-序号，例如wz-tygr-001",
      source: "问真八字", // 固定值
      screenshotRef: "截图文件名，例如wenzhen-tygr-001.png，需保存在docs/shensha-samples/目录",
      input: {
        solarTime: "问真输入的公历出生时间，格式YYYY-MM-DD HH:mm:ss",
        gender: "男/女",
        longitude: 经度，默认东八区填120，真太阳时按实际输入的经度填,
        latitude: 纬度，默认填39，真太阳时按实际输入的纬度填,
        useTrueSolarTime: true/false, // 问真中真太阳时开关状态
        useDst: true/false // 问真中夏令时开关状态
      },
      expect: {
        pillars: { // 问真排盘结果的四柱，必须和截图完全一致
          year: "年柱干支，例如己巳",
          month: "月柱干支，例如丙子",
          day: "日柱干支，例如丙寅",
          hour: "时柱干支，例如甲午"
        },
        spirits: [ // 该命例命中的该神煞信息
          {
            name: "神煞名称，和spiritName一致",
            location: "命中的柱位：年柱/月柱/日柱/时柱",
            branch: "命中的地支，例如巳"
          }
        ]
      }
    }
  ]
}
```

### 填写注意事项
1. 不命中样本的`expect.spirits`为空数组即可，其他字段和命中样本要求一致
2. 经纬度如果没有手动调整，统一填`longitude: 120, latitude: 39`
3. 截图必须完整包含四柱和神煞显示区域，确保可以核对结果
4. 问真没有显示的神煞不要自行添加，哪怕传统命理认为应该命中，也以问真实际显示为准

## 填写示例（天乙贵人命中样本）
```js

          month: "丙子",
          day: "丙寅",
          hour: "甲午"
        },
        spirits: [
          {
            name: "天乙贵人",
            location: "时柱",
            branch: "午"
          }
        ]
      }
    }
  ]
}
```

## 验收标准
### 核心红线（必须严格遵守）
**没有问真截图和对应结构化样本的神煞，绝对不进入计算引擎开发阶段**，所有规则100%以问真实际显示为准，禁止用传统命理资料自行推断。

### 样本要求
1. 10个神煞每个至少3-5个有效样本，必须覆盖：命中/未命中场景、年/月/日/时不同柱位命中场景
2. 所有样本都有对应截图，截图可清晰核对四柱和神煞结果，截图统一存放于`docs/shensha-samples/`目录
3. 样本格式校验通过：每录完一批执行`npm run sample:shensha`无格式错误
4. 就绪度校验通过：所有神煞采集完成后，`summarizeWenZhenShenShaReadiness`返回`ready: true`，10个神煞全部就绪
5. 所有神煞规则均来自问真实际显示，无自行推断、补全内容

---

## 校验方式
采集完成后，在项目根目录执行：
```bash
npm run sample:shensha
```
脚本会自动校验所有样本格式，输出就绪度统计，只有全部通过才能进入神煞引擎开发阶段。
