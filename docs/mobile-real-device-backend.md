# 手机真机连接后端说明

## 目标

手机真机测试时，小程序必须连接后端服务，才能使用完整的排盘、农历、闰月、节气和真太阳时能力。

## 当前边界

- 电脑预览可以使用 `http://127.0.0.1:8787`。
- 手机真机不能使用 `127.0.0.1` 或 `localhost` 访问电脑后端。
- 真机测试需要使用电脑在同一 Wi-Fi 下的局域网地址，例如 `http://192.168.1.23:8787`。
- 正式发布必须换成备案、可信、可访问的 HTTPS 域名。

## 开发测试步骤

1. 电脑启动后端服务。
2. 手机和电脑连接同一个 Wi-Fi。
3. 查询电脑局域网 IP。
4. 把小程序运行配置里的 `baziApi.baseUrl` 改为 `http://电脑局域网IP:8787`。
5. 在手机真机点击排盘，确认结果页显示后端返回的命盘、节气和农历信息。

## 必查接口

- `GET /health`：确认后端服务在线。
- `GET /bazi/calendar/coverage`：确认当前农历、闰月和节气数据覆盖范围。
- `POST /bazi/calculate`：正式排盘入口。

## 不能做的事

- 不要让手机真机继续使用 `127.0.0.1`。
- 不要把完整农历和节气大数据包硬塞进小程序包体。
- 不要在前端白名单补样例来假装排盘准确。
- 不要绕过后端直接在页面里写死命盘结果。

## 验收标准

- 电脑预览能连本机后端。
- 手机真机能连局域网后端。
- 手机真机无法连接时，要能明确判断是不是使用了本机地址。
- 后端仍然是排盘、农历、闰月、节气和真太阳时的唯一权威计算层。
