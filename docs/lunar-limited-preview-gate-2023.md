# 2023 HKO 受限农历预览前置门禁

## 当前结论

2023 HKO 候选包已经生成，但仍不能开放给用户使用。

原因不是数据没有价值，而是预览入口开放前必须先具备两类保护：

1. 用户必须看到清楚的范围警示。
2. 超出范围时必须返回稳定、可解释的越界错误。

## 当前状态

- 预览入口：关闭。
- runtime：关闭。
- manifest 注册：关闭。
- 完整农历能力：否。

## 必须显示的警示语义

所有未来预览入口至少必须表达：

- 仅支持 2023 年。
- 来源是 HKO / 香港天文台。
- 这不是完整农历历法能力。

建议文案：

```text
当前仅为 2023 年 HKO / 香港天文台农历数据预览，非完整农历历法能力；超出 2023 年范围的农历输入不会计算。
```

## 越界错误要求

错误码固定为：

```text
LUNAR_DATE_OUTSIDE_LIMITED_RUNTIME_SCOPE
```

错误内容必须包含：

```text
inputDate
supportedStartDate
supportedEndDate
completeLunarCalendar
calendarDataVersion
```

必须覆盖至少两类越界：

1. 2023-01-01 之前。
2. 2023-12-31 之后。

## 禁止事项

1. 不得注册 `code/data-packs/lunar/manifest.json`。
2. 不得接入正式八字排盘入口。
3. 不得宣称完整农历能力。
4. 不得在没有 UI 警示的情况下开放预览入口。
5. 不得在没有结构化越界错误的情况下注册候选包。

## 下一步

下一轮可以开始实现“受限预览入口”，但必须继续保持：

- 默认关闭。
- 有明显范围警示。
- 有稳定越界错误。
- 不影响正式公历排盘。
