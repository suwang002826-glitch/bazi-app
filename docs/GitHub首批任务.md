# GitHub 首批任务

把项目上传到 GitHub 后，建议先创建这些 Issue。每次只做一张卡，不要一次合并多个方向。

## Issue 1：确认排盘规则

目标：
整理第一版采用的排盘规则，确认哪些已采用，哪些待确认。

验收标准：
1. `排盘规则.md` 中列出子时换日、节气换月、真太阳时、闰月、大运起运。
2. 每个待确认规则都有负责人。
3. 不改代码，只确认规则。

建议分支：
`docs/confirm-rules`

## Issue 2：补充 20 个验收样例

目标：
建立第一批排盘准确性样例。

验收标准：
1. `验收样例.md` 至少有 20 个样例。
2. 每个样例都有来源。
3. 至少包含立春、节气、子时、闰月场景。

建议分支：
`docs/acceptance-samples`

## Issue 3：复核四柱计算

目标：
用验收样例检查当前排盘结果。

验收标准：
1. 至少对照 10 个样例。
2. 不一致结果要记录原因。
3. 如需改算法，先更新 `排盘规则.md`。

建议分支：
`feature/verify-pillars`

## Issue 4：优化保存记录

目标：
让保存记录更适合真实使用。

验收标准：
1. 可以查看、载入、删除保存记录。
2. 保存记录显示姓名、四柱、保存时间。
3. 手机端记录卡片不溢出。

建议分支：
`feature/save-records`

## Issue 5：制作分享图片

目标：
生成一张适合发给朋友的命盘图片。

验收标准：
1. 图片包含姓名、四柱、基础信息。
2. 图片中文字清晰。
3. 手机端可以保存或复制图片。

建议分支：
`feature/share-image`

## Issue 6：部署测试版

目标：
把项目发布成可访问的网址。

验收标准：
1. 可以通过公开网址访问。
2. 手机浏览器打开正常。
3. README 记录部署地址。

建议分支：
`chore/deploy-preview`
