# 排盘规则（v0.2）

以下为当前稳定核对口径，代码与验收脚本必须与此一致。

## 核心规则

- 换年：立春精确时刻为界，交节前属上一年柱，交节时刻及之后属新年柱。
- 换月：按十二节（立春、惊蛰、清明、立夏、芒种、小暑、立秋、白露、寒露、立冬、大雪、小寒）精确时刻为界，顺序对应寅月到丑月。
- 换日：按 23:00 整换日，23:00及之后按次日日柱；0:00 不换日。
- 子时：
  - 默认不区分早晚子，23:00-01:00 为子时，日柱/时柱按次日五鼠遁。
  - `useEarlyLateZi=true` 时，23:00-00:00 夜子：日柱保当日，时干按次日；00:00-01:00 早子：日柱按次日，时干按次日。
- 真太阳时：
  - 默认关闭。
  - 开启后按东经120°基准修正经度偏移（每度4分钟）+ 均时差（Equation of Time）。
  - 经纬度空值默认 `120.0` / `30.0`，不做反向地理编码。
- 夏令时：
  - 默认“自动识别”。
  - 仅 1986-1991 年按官方起止时间处理，区间内先减 1 小时，1992 年及以后不执行夏令时。
- 农历：支持农历输入、`isLeapMonth` 闰月参数；规则以公历转换结果参与统一四柱计算。
- 字段规范：
  - 天干：`甲乙丙丁戊己庚辛壬癸`
  - 地支：`子丑寅卯辰巳午未申酉戌亥`
  - 输入时间：`YYYY-MM-DD HH:mm:ss`
  - 输出四柱：`{year,month,day,hour}`（每项干支字符串）

## 默认值（工程内统一）

- `defaultTrueSolarTime: false`
- `defaultDST: true`
- `defaultEarlyLateZi: false`
- `defaultLongitude: 120.0`
- `defaultLatitude: 30.0`

## 验收基线

目前验收基线固定为：`scripts/test-cases.js`（共 30 条），执行方式：

```bash
node scripts/verify.js
```

基线字段采用统一 JSON 格式：`{ case_id, scene, input, expect }`，输入支持 `solar_time` 或 `birthDate+birthTime`，并支持：

- `use_true_solar`
- `use_dst`
- `use_early_late_zi`

每条用例必须 100% 命中后再进入下一步单元开发。
