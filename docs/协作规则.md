## 固定流程

每个任务都按这个顺序推进：

```text
创建 Issue
确认负责人
创建任务分支
完成修改
本地预览或说明验证方式
创建 Pull Request
另一人检查
合并到 main
```

- 八字排盘准确性：`feature/bazi-accuracy`
- 八字结果页 UI：`feature/bazi-ui-song-style`
- 文档规则整理：`docs/rules`
- 修复问题：`fix/问题简述`

## Pull Request 检查

合并前至少检查：

- 是否只做了当前 Issue 的内容
- 是否影响已有页面
- 是否可以在微信开发者工具预览
- 是否更新了必要文档
- 是否有排盘样例或说明
- 产品与验收负责人是否确认通过

## 每日同步

每天只同步四件事：

1. 今天做哪个 Issue？
2. 谁负责开发？
3. 谁负责验收？
4. 是否已经合并到 `main`？
