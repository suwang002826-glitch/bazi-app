# 协作规则

## 共同目标

两个人都围绕当前 GitHub 仓库协作，不各自做独立版本。当前仓库地址：

```text
https://github.com/suwang002826-glitch/bazi-app
```

当前主要开发目录：

```text
code/
```

## 固定流程

每个任务都按这个顺序推进：

```text
创建 Issue
确认负责人
创建任务分支
完成修改
本地预览或说明验证方式
创建 Pull Request
另一人检查
合并到 main
```

## 分工

产品与验收负责人：

- 产品方向
- 中文界面
- 使用体验
- 排盘准确性验收
- 测试反馈
- 验收样例整理

开发负责人：

- 代码开发
- 规则库补全
- 微信开发者工具调试
- Bug 修复
- Pull Request 实现说明

## 分支命名

- 八字排盘优化：`feature/bazi-plate`
- 六爻页面优化：`feature/liuyao-ui`
- 奇门规则优化：`feature/qimen-rules`
- 文档规则整理：`docs/rules`
- 修复问题：`fix/问题简述`

## Pull Request 检查

合并前至少检查：

- 是否只做了当前 Issue 的内容
- 是否影响已有页面
- 是否可以在微信开发者工具预览
- 是否更新了必要文档
- 是否有排盘样例或说明

## 每日同步

每天只同步四件事：

1. 今天做哪个 Issue？
2. 谁负责开发？
3. 谁负责验收？
4. 是否已经合并到 `main`？
