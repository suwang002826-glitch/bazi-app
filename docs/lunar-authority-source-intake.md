# 权威万年历资料接收标准

## 是否需要你找万年历

需要，但要找“可追溯、可复核、可批量整理”的资料。

我们现在已有 HKO 2023 数据候选包，但它只覆盖 2023 公历年。要继续扩展年份、闰月和完整农历能力，就需要更多权威来源。

## 最推荐的资料

优先级从高到低：

1. 官方或天文机构发布的 CSV、TXT、Excel、结构化表格。
2. 官方 PDF 或权威出版社纸质万年历。
3. 专业软件导出的批量对照表。
4. 普通网页或截图，只能当辅助参考，不能单独作为权威来源。

## 最好给我的格式

最理想：

```text
CSV / Excel / TXT
```

每行包括：

```text
公历日期, 农历年, 农历月, 农历日, 是否闰月, 来源名称, 页码或行号
```

也可以给 PDF 或纸质书照片，但需要额外提供：

```text
书名, 出版社, 版次, 年份, 页码, 拍照日期
```

## 不建议直接使用的资料

这些资料可以参考，但不能直接进正式数据包：

1. 只有截图，没有来源。
2. 只写“网上查的”，没有网址或出版信息。
3. 闰月标注不清楚。
4. 只能查单日，不能批量核对。
5. 软件结果没有版本、设置和导出日期。

## 当前处理原则

你给资料后，我们先做三步：

1. 记录来源。
2. 转成草案数据。
3. 和现有 HKO / 样例 / 后续独立来源交叉校验。

只有经过复核的数据，才可能进入候选包。正式 runtime 还需要单独审批。
