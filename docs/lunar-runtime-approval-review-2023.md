# 2023 HKO 农历草案 runtime approval 评审

当前结论：不批准 runtime。

可以进入下一阶段的设计工作：`2023 公历年受限 runtime`。

## 为什么不直接批准

- 当前复核仍是 HKO CSV 同源复核，不是独立第三方来源复核。
- 当前数据覆盖完整公历年 2023，但不是完整农历年库。
- 用户界面还没有范围提示。
- runtime manifest 还没有单独评审。

## 可以设计的受限能力

如果后续批准，只能声明：

- 支持 HKO 2023 公历年范围内的农历转换；
- 不支持完整历史农历库；
- 不支持完整农历年覆盖声明；
- 超出范围必须结构化报错。

## 必须具备

- `completeLunarCalendar: false`
- UI 范围提示
- 越界结构化错误
- 单独 runtime manifest 注册评审
- 明确版本号：`lunar-data-pack@2026.07.01-hko-2023-limited.1`

## 禁止宣传

- 禁止宣传“完整农历排盘已上线”。
- 禁止宣传“全历史农历转换”。
- 禁止宣传“独立权威已验证”。

这一步仍然服务于产品核心宗旨：精准、稳定、专业。
