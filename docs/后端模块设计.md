# 八字排盘后端/规则模块结构设计

## 定位

当前项目是微信小程序，主要代码在 `code/`。第一版不接服务器，不引入 TypeScript、Node、Python、Java/Kotlin 等新技术栈。这里的“后端”指小程序内部的规则层、数据结构、校验框架和未来接口预留。

当前阶段只做 `feature/bazi-accuracy` 的技术准备，不正式改八字算法。

## 模块职责

### `inputNormalizer.js`

统一八字输入，把页面表单转换成算法内部使用的标准结构。

负责：

- 公历 / 农历 / 四柱输入模式识别
- 日期、时间、性别、出生地字段标准化
- 空值、非法日期、非法时辰的错误提示
- 保留原始输入，方便排查问题

### `ruleConfig.js`

集中管理排盘规则口径。等 #4 确认后，把规则写成配置，不要散落在页面代码中。

负责：

- 是否按节气换月
- 子时换日口径
- 真太阳时是否默认启用
- 起运规则
- 大运、流年、流月口径
- 神煞取法优先级

### `calendarAdapter.js`

日历转换适配层。后续如果替换权威历法库，只改这里。

负责：

- 公历转农历
- 农历转公历
- 闰月处理
- 日期合法性判断
- 节气数据入口预留

### `solarTerm.js`

节气与换月规则层。

负责：

- 根据出生时间判断所属节气区间
- 月柱换月判断
- 节气交接前后边界处理
- 等 #4 明确后再落地具体算法

### `trueSolarTime.js`

真太阳时换算层。

负责：

- 出生地经度输入
- 标准时间到真太阳时的换算接口
- 经度来源标记
- 默认是否启用由 #4 决定

### `pillarEngine.js`

四柱计算入口。当前先只设计接口，不正式替换现有算法。

负责：

- 年柱、月柱、日柱、时柱计算入口
- 天干地支、藏干、十神、纳音、旬空等结果组织
- 调用 `ruleConfig`、`calendarAdapter`、`solarTerm`、`trueSolarTime`

### `fortuneCycle.js`

起运、大运、流年、流月预留层。

负责：

- 起运方向
- 起运年龄
- 大运列表
- 流年、流月、流日、流时接口
- 规则未定前只保留结构

### `validation.js`

验收样例校验层。

负责：

- 读取验收样例
- 调用排盘入口
- 比对期望四柱和当前结果
- 输出通过 / 不通过 / 差异原因

### `index.js`

统一导出模块能力，页面只从这里调用。

## 实施顺序

1. 等 #4 明确规则口径。
2. 等 #5 给出第一批验收样例。
3. 在 `feature/bazi-accuracy` 中先加模块骨架和校验入口。
4. 用样例证明当前算法哪里不一致。
5. 只针对不一致项逐步修正。

## 禁止事项

- 不在规则未确认前重写算法。
- 不把规则写散在页面事件里。
- 不引入新技术栈。
- 不为了 UI 美化改动排盘逻辑。
