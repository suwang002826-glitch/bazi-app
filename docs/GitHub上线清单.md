# GitHub 上线清单

当前本地项目已经准备好。由于本机未安装 Git / GitHub CLI，Codex 的 GitHub 连接也尚未授权，所以线上仓库、邀请朋友、创建线上 Issue 这几步需要先完成授权。

## 需要你完成一次的事项

1. 注册或登录 GitHub。
2. 创建一个私有仓库，建议命名为 `bazi-app`。
3. 在 Codex 里连接 GitHub 账号。
4. 提供仓库地址，例如：`用户名/bazi-app`。
5. 提供朋友的 GitHub 用户名，用于邀请协作。

## 授权后可继续由 Codex 代办

- 把本项目文件上传到 GitHub 仓库。
- 创建首批 Issue。
- 创建任务分支。
- 创建 Pull Request。
- 帮你们写 PR 描述。
- 检查朋友是否有仓库权限。
- 按 Issue 逐个开发功能。

## 仓库建议设置

- 仓库先设为 Private。
- `main` 分支设为稳定版本。
- 所有功能通过 Pull Request 合并。
- 每个 Pull Request 至少由另一个人检查。
- 熟悉流程后开启 `main` 分支保护。

## 首次上传后的建议操作

1. 创建 Issue 1：确认排盘规则。
2. 创建 Issue 2：补充 20 个验收样例。
3. 创建 Issue 3：复核四柱计算。
4. 两个人先完成文档和样例，再继续扩展功能。
