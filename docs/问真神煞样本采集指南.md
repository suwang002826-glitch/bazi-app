# 问真神煞样本采集指南

## 目标

本阶段只收集并验收问真八字的神煞标准样本，不实现任何神煞算法，不做自定义解释。

所有神煞规则必须满足两个条件后才能进入开发：

- 有明确的计算规则描述，说明取年干、日干、年支、日支、月令或其他依据。
- 有问真八字实际排盘命例截图和结构化期望结果，且可复现。

## 单条样本必须包含

```js
{
  spiritName: "天乙贵人",
  rule: {
    basis: "以问真八字实际显示为准，例如按日干查贵人地支",
    calculation: "记录问真页面命中的柱位和地支，不自行外推"
  },
  cases: [
    {
      caseId: "wz-tygr-001",
      source: "问真八字",
      screenshotRef: "wenzhen-tygr-001.png",
      input: {
        solarTime: "1990-01-01 12:00:00",
        gender: "男",
        longitude: 120,
        latitude: 39,
        useTrueSolarTime: false,
        useDst: false
      },
      expect: {
        pillars: {
          year: "己巳",
          month: "丙子",
          day: "丙寅",
          hour: "甲午"
        },
        spirits: [
          {
            name: "天乙贵人",
            location: "年柱",
            branch: "巳"
          }
        ]
      }
    }
  ]
}
```

## 采集口径

- `source` 固定写 `问真八字`。
- `solarTime` 必须是问真输入页使用的出生时间。
- 经纬度未知时按项目默认口径填 `longitude: 120`、`latitude: 39`。
- 必须记录真太阳时、夏令时开关状态。
- `pillars` 必须填写问真结果中的四柱，用于确认样本不是录入错误。
- `spirits` 只填问真页面明确显示的神煞命中结果。
- 问真没有显示的神煞不要自行补。

## 开发门槛

任一神煞进入开发前，必须先通过 `validateWenZhenShenShaSample` 结构校验。

全部第一批神煞进入计算实现前，必须先通过 `summarizeWenZhenShenShaReadiness`，确保 10 个目标神煞均已有问真样本。

## 模板生成

可执行：

```bash
npm run sample:shensha
```

生成第一批神煞的待采集 JSON 模板。
